<task type="auto">
  <name>Task 1: Install dependencies and build dist/index.js</name>
  <files>node_modules/, dist/index.js</files>
  <action>
Run `npm install` to install devDependencies (vite, typescript, @types/react), then run `npm run build` to produce dist/index.js.

```bash
npm install
npm run build
```

After the build completes, verify the output is correct:
1. `dist/index.js` must exist
2. First lines must contain `import ... from "data:text/javascript,..."` (not bundled React)
3. File size should be well under 10KB — if it's larger, React was accidentally bundled and the vite.config.ts must be fixed before proceeding

If the build fails with a TypeScript error, check src/index.tsx and tsconfig.json from Plan 01 are correct. Do NOT modify vite.config.ts unless the error explicitly points to it.

If dist/index.js first lines do NOT contain `data:text/javascript`, do NOT commit — the React externalization is broken. Check that `rollupOptions.external` in vite.config.ts contains `['react', 'react-dom', 'react/jsx-runtime']` and that `output.paths` maps each to its data: URL.
  </action>
  <verify>
    <automated>test -f dist/index.js &amp;&amp; head -2 dist/index.js | grep -q "data:text/javascript" &amp;&amp; wc -c dist/index.js | awk '{if ($1 > 102400) {print "FAIL: dist/index.js too large (" $1 " bytes) — React may be bundled"; exit 1} else print "PASS: " $1 " bytes"}' &amp;&amp; echo "BUILD OUTPUT VALID"</automated>
  </verify>
  <done>dist/index.js exists, starts with data: URL imports (React externalized), and is under 100KB</done>
</task>

<task type="auto">
  <name>Task 2: Commit dist/index.js to the repository</name>
  <files>dist/index.js</files>
  <action>
Initialize git if needed, then stage and commit dist/index.js. This is the critical step that makes Ship Studio loadable — the plugin loader fetches dist/index.js directly from the git repository.

Check git status first. If this is a fresh repo with no commits, initialize and make the initial commit covering ALL project files. If the repo already has commits, add dist/index.js and all other untracked files from Plan 01 (src/index.tsx, plugin.json, package.json, tsconfig.json, vite.config.ts, .gitignore):

```bash
# Check current git status
git status

# Stage all plugin files (not node_modules — .gitignore handles that)
git add dist/index.js src/index.tsx plugin.json package.json package-lock.json tsconfig.json vite.config.ts .gitignore

# Commit
git commit -m "feat(01-foundation): scaffold plugin skeleton with working build"
```

CRITICAL: Verify dist/index.js is tracked by git after the commit:
```bash
git show HEAD --name-only | grep "dist/index.js"
```

If dist/index.js is NOT in the commit (e.g., .gitignore is wrong), fix .gitignore to remove any dist/ entry and recommit.
  </action>
  <verify>
    <automated>git show HEAD --name-only | grep -q "dist/index.js" &amp;&amp; git status dist/index.js | grep -q "nothing to commit\|dist/index.js" &amp;&amp; echo "dist/index.js is committed to git"</automated>
  </verify>
